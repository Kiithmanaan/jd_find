# 前端产品设计

## 1. 信息架构

第一阶段前端分为：

- 登录页。
- JobProfile 列表页。
- JobProfile 创建/编辑/确认页。
- JobProfile 详情页。
- SearchRun 详情页。
- 候选人汇总页。
- AI 审计详情页或弹窗。

下一步优先版本与重新评估相关页面。

代码实现前置口径：

- 第一阶段先用低保真锁定交互，不要求高保真视觉稿。
- 高保真稿可在功能实现后继续补齐。
- 页面实现优先满足演示和验收，不先做复杂视觉定制。

## 2. 登录页

能力：

- 邮箱 + 密码登录。
- Web JWT 过期后提示重新登录。
- 不开放注册。

状态：

- 登录中。
- 登录失败。
- Token 过期。

## 3. JobProfile 列表页

展示字段：

- 岗位名称。
- 当前版本号。
- 状态。
- 创建者。
- 最近更新时间。
- SearchRun 数量。

操作：

- 新建岗位画像。
- 打开详情。
- 复制岗位画像。

## 4. JobProfile 创建/编辑/确认页

核心区域：

- JD 原文。
- 搜索条件。
- 硬性条件规则编辑器。
- 软性条件自然语言文本。
- AI 生成的岗位画像建议。

确认行为：

- AI 不自动确认岗位画像。
- 用户点击确认后生成新的 confirmed version。
- 已确认版本不可编辑。
- 编辑已确认画像时创建草稿版本。

## 5. JobProfile 详情页

展示：

- 当前 confirmed version。
- 版本历史。
- 当前版本硬性条件。
- 当前版本软性条件文本。
- 当前版本匹配 prompt。

操作：

- 启动插件 `SearchRun`。
- 触发历史候选人批量重新评估。
- 查看候选人汇总 Tab。

候选人汇总 Tab：

- 当前版本已评估。
- 未按当前版本评估。

低保真结构：

- 顶部：岗位名称、当前版本号、确认状态、最近更新时间。
- 左侧主区：当前版本画像、硬性条件、软性条件文本、匹配 prompt。
- 右侧操作区：启动 SearchRun、手动重新评估、查看版本历史。
- 下方 Tab：当前版本已评估候选人、未按当前版本评估候选人。
- 版本历史只展示画像本身，不支持切换评估版本。

## 6. SearchRun 详情页

展示：

- 状态。
- 原始提交数量。
- 目标数量。
- 候选人处理进度。
- 候选人列表。
- AI 审计入口。

状态展示：

- `Running`：展示提交进度。
- `Completed`：展示完成时间和候选人结果。
- `Failed`：展示失败原因。
- `Cancelled`：展示取消原因。
- `Interrupted`：仅用于历史 Mock/Adapter 风控中止场景。

低保真结构：

- 顶部：SearchRun 状态、目标原始候选人数量、已提交数量、完成条件提示。
- 操作区：Running 状态展示取消入口；非 Running 状态禁用取消。
- 主列表：候选人处理结果。
- 侧边或底部：AI 审计入口和批次处理信息。

## 7. 候选人列表

直接展示字段：

- 姓名。
- 当前职位。
- 城市。
- 学历。
- 工作年限。
- 行业。
- 匹配分。
- 推荐结论。
- 求职意向。
- 活跃度。
- 来源平台。
- SourceLead URL。
- fallbackClues。
- 简历附件下载入口。

交互：

- 点击匹配分打开悬浮详情弹窗。
- 点击附件下载简历。

低保真结构：

- 列表按匹配分和推荐结论排序展示最新结果。
- 推荐结论独立列展示。
- 匹配分为可点击文本或按钮，打开详情悬浮弹窗。
- 附件入口只展示插件已上传的简历附件。
- 无附件时展示空状态，不提供前端手动上传。

## 8. 匹配详情悬浮弹窗

展示顺序：

1. 推荐结论。
2. 匹配分。
3. 合适点。
4. 不合适点。
5. 风险点。
6. 硬筛 trace。
7. 软性匹配 trace。

空状态：

- 无推荐结论：显示“暂无推荐结论”。
- 无不合适点：显示“暂无明显不合适点”。
- 无风险点：显示“暂无明显风险点”。
- 无 trace：显示“暂无 trace 记录”。

低保真结构：

- 触发方式：在 SearchRun 详情页候选人列表点击匹配分打开。
- 弹窗顶部展示候选人姓名、推荐结论和匹配分。
- 中部展示合适点、不合适点、风险点三组列表。
- 底部展示硬筛 trace 和软性匹配 trace。
- 关闭后回到候选人列表，不改变列表筛选状态。

## 9. AI 审计详情

展示：

- provider。
- model。
- prompt version。
- agent version。
- duration。
- candidateIds。
- inputSnapshot。
- outputSnapshot。
- errorType。
- errorMessage。

## 10. 目标数量文案

启动 SearchRun 时展示：

- 字段名：目标原始候选人数量。
- 默认值：200。
- 范围：10-500。
- 提示：达到目标数量或插件上报完成后，SearchRun 可进入完成条件。
