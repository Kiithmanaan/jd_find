# 硬性条件配置说明

## 1. 第一阶段维度

第一阶段硬性条件维度：

- 全文关键词。
- 城市。
- 行业。
- 学历。
- 最低工作年限。

## 2. 规则结构

硬筛采用两段式：

1. 淘汰规则。
2. 合格规则。

判定顺序：

1. 命中任一淘汰规则，直接淘汰。
2. 未命中淘汰规则后执行合格规则。
3. 命中任一合格规则，硬筛通过。
4. 未命中淘汰规则且未命中任何合格规则，淘汰。

规则表达：

- 支持多条淘汰规则。
- 支持多条合格规则。
- 多条淘汰规则之间为 OR。
- 多条合格规则之间为 OR。
- 单条规则只支持单层条件。
- 单条规则内部选择 `AND` 或 `OR`。
- 第一阶段不支持复杂条件树。

## 3. 配置表维护

硬性条件配置通过数据库 seed 维护，前端只读取配置。

维度配置字段：

- `key`：维度唯一键。
- `label`：展示名。
- `valueType`：`text` / `number` / `option`。
- `supportedMatchModes`：支持的匹配模式。
- `allowMultiple`：是否允许多值。

选项配置字段：

- `dimensionKey`。
- `value`。
- `label`。
- `aliases`。
- `rank`：用于学历等级比较。

## 5. 匹配规则

全文关键词：

- 支持精确包含。
- 支持归一化包含。
- 支持多个关键词。
- 多关键词支持任一命中和全部命中。
- 不做语义匹配，语义匹配归软性条件。

归一化包含忽略：

- 大小写。
- 半角空格和全角空格。
- 常见中英文标点。
- 全角半角差异。

城市：

- 标准值和别名均可命中。
- 多城市任一命中。
- 支持不限城市。

行业：

- 标准值和别名均可命中。
- 多行业任一命中。

学历：

- 按 rank 比较。
- 候选人学历 rank 大于等于要求 rank 视为满足。

最低工作年限：

- 候选人年限大于等于要求年限视为满足。
- 年限未知第一阶段视为不满足。

## 6. Trace 展示

硬筛 trace 需要保存并展示：

- 命中的规则类型：淘汰 / 合格。
- 命中的规则名称。
- 命中的条件。
- 候选人实际值。
- 配置期望值。
- 结果：通过 / 淘汰。

前端展示位置：

- 候选人匹配详情弹窗。
- SearchRun 详情页候选人列表的硬筛状态入口。
- AI 审计详情中可关联硬筛输入快照。
