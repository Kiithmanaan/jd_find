## 10. 待后续重点复核的设计风险

- 当前候选人数据全部保存且不脱敏，正式上线前需要复核数据合规风险。
- Web 用户第一阶段可看所有数据，正式多人使用前需要复核权限模型。
- 主工程与浏览器插件的边界需要严格保持，避免主工程承担抓取风控策略。
- 插件增量提交会改变当前批处理 SearchRun 模型，需要单独设计队列和缓冲策略。
- JobProfileVersion、重新评估和候选人汇总视图会显著改变现有领域模型，需要优先做架构调整。
